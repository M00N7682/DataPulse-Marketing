# 디지털 마케팅 캠페인 데이터 기반 ROAS 예측 연구 계획

## 1. 연구 목적

- 실제 디지털 마케팅 캠페인 데이터를 기반으로 고객 전환율 예측 모델 구축
- CTGAN 및 Tabular Diffusion을 이용한 합성 마케팅 데이터 생성
- 합성데이터를 결합한 Hybrid 학습으로 예측 성능, 일반화 성능, 효율성 검증
- 개인정보보호 규제 환경에서의 마케팅 의사결정 자동화 가능성 평가

## 2. 데이터셋 특성 분석

### 현재 데이터셋 구조 (8,002 레코드 × 20 변수)

**타겟 변수:**
- `Conversion` (0/1): 고객 전환 여부 (이진 분류)

**주요 마케팅 성과 지표:**
- `AdSpend`: 광고비
- `ClickThroughRate`: 클릭률
- `ConversionRate`: 전환율
- `WebsiteVisits`: 웹사이트 방문 수
- `PagesPerVisit`: 방문당 페이지 수
- `TimeOnSite`: 사이트 체류 시간

**고객 특성:**
- `Age`, `Gender`, `Income`: 인구통계학적 정보
- `PreviousPurchases`: 이전 구매 이력
- `LoyaltyPoints`: 로열티 포인트

**캠페인 정보:**
- `CampaignChannel`: Social Media, Email, PPC, SEO, Referral
- `CampaignType`: Awareness, Conversion, Retention, Consideration
- `AdvertisingPlatform`, `AdvertisingTool`: 광고 플랫폼 정보

**행동 데이터:**
- `SocialShares`, `EmailOpens`, `EmailClicks`: 참여도 지표

### ROAS 계산 방식 정의
```
ROAS = (전환가치) / (광고비)
전환가치 = Income × ConversionRate × (채널별 가중치)
또는
전환가치 = Conversion × 평균구매가격 (추정)
```

## 3. 데이터 분석 계획

### (1) 탐색적 데이터 분석(EDA)
**목표:** 데이터 구조, 변수 간 관계, 이상치·결측치를 파악하여 모델링 방향 결정

**수행 내용:**
- 변수 타입별 기본 통계량 분석 (df.describe())
- 결측률·이상값 확인 (isna().sum(), IQR 기준 이상치 제거)
- 채널별 광고비-전환율-고객특성 상관관계 시각화
- 캠페인 유형별 성과 분석 (Awareness vs Conversion vs Retention vs Consideration)
- 고객 세그먼트별(연령, 소득, 성별) 전환율 차이 분석

### (2) 데이터 전처리 및 피처 엔지니어링
**핵심 파생 변수 생성:**
```python
# ROAS 관련 파생 변수
roas_estimated = (Income * ConversionRate) / AdSpend
cost_per_conversion = AdSpend / (ConversionRate + 1e-8)
efficiency_score = ClickThroughRate * ConversionRate

# 고객 가치 지표
customer_lifetime_value = Income * (LoyaltyPoints / 1000) * PreviousPurchases
engagement_score = (SocialShares + EmailOpens + EmailClicks) / 3
website_engagement = PagesPerVisit * TimeOnSite

# 채널 효율성
channel_spend_ratio = AdSpend / Income
conversion_efficiency = Conversion / WebsiteVisits

# 범주형 변수 인코딩
channel_encoded = pd.get_dummies(CampaignChannel)
campaign_type_encoded = pd.get_dummies(CampaignType)
```

**전처리 단계:**
1. 로그 변환 (AdSpend, Income 등 치우친 분포)
2. 표준화/정규화 (Z-score, Min-Max)
3. 범주형 변수 인코딩 (One-hot, Label encoding)
4. 이상치 처리 (IQR 방법)

**데이터 분할:**
- 학습/검증/테스트: 70:15:15 (랜덤 분할)
- 채널별 홀드아웃 셋 구성 (콜드스타트 시나리오용)

## 4. 합성데이터 생성 계획

### (1) 생성 모델 선택
**CTGAN (Conditional Tabular GAN):**
- 범주형/연속형 혼합 데이터에 강점
- 조건부 생성으로 채널별 특성 유지

**Tabular Diffusion (TabDDPM):**
- 노이즈 제거 기반으로 안정적 분포 복원
- 더 나은 상관관계 보존

### (2) 실험 절차
1. **피처 선택:** 핵심 마케팅 변수 선별
   - AdSpend, ClickThroughRate, ConversionRate, Income
   - CampaignChannel, CampaignType, Age, Gender
   - WebsiteVisits, TimeOnSite, PreviousPurchases

2. **모델 학습:**
   - CTGAN: 500 epochs, batch_size=500
   - TabDDPM: 1000 steps, lr=1e-4

3. **합성 데이터 생성:**
   - 생성량: 실제 데이터의 100% (8,000개)
   - 채널별 조건부 생성으로 균형 유지

### (3) 품질 평가 지표
**통계적 유사성:**
- Kolmogorov-Smirnov Distance (단변량 분포)
- Wasserstein Distance (다변량 분포)
- Jensen-Shannon Divergence

**구조적 유사성:**
- Correlation Preservation Ratio
- Mutual Information Score
- Principal Component Analysis 비교

## 6. 성능 및 효율성 평가 계획

### (1) 분류 성능 지표
**주요 지표:**
- **Accuracy:** 전체 정확도
- **Precision/Recall:** 전환 예측 정밀도/재현율
- **F1-Score:** 균형 성능 지표
- **AUC-ROC:** 분류 성능 종합 평가

**비즈니스 지표:**
- **Cost per True Positive:** 실제 전환 고객 획득 비용
- **Marketing Efficiency:** 마케팅 효율성 개선도

### (2) 합성데이터 품질 지표
- **Fidelity:** 통계적 유사성 (KS Test < 0.1)
- **Utility:** 예측 성능 유지율 (> 95%)
- **Privacy:** 개인정보 유출 위험도

### (3) 효율성 지표
```python
# 콜드스타트 성능 향상률
coldstart_improvement = (F1_hybrid - F1_real) / F1_real * 100

# 데이터 증강 효과
data_augmentation_effect = (Performance_augmented - Performance_baseline) / Performance_baseline

# 비용 효율성
cost_efficiency = F1_improvement / (training_time_increase + data_generation_cost)
```
